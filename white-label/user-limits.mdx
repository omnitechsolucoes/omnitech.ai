Gerencie limites e restrições individuais de usuários além dos limites padrão dos planos. Isso permite personalizar o acesso de usuários específicos sem a necessidade de alterar o plano contratado.

## Visão Geral

Os limites de usuário oferecem controle granular sobre cada usuário da sua plataforma **OmniTech IA White Label**. Com eles, é possível sobrescrever os limites do plano para usuários específicos, concedendo mais ou menos recursos conforme a necessidade operacional ou comercial.

## Tipos de Limites Disponíveis

### Limites de Quantidade
**Restrições numéricas sobre o uso de recursos:**

<AccordionGroup>
  <Accordion title="Assistentes" icon="robot">
    **Finalidade:** Número máximo de assistentes de IA que o usuário pode criar
    
    **Sugestão de Planos Personalizados:**
    - Gratuito: 1 assistente (recomendado)
    - Básico: 2–3 assistentes
    - Profissional: 5–10 assistentes
    - Enterprise: Ilimitado
  </Accordion>
  
  <Accordion title="Campanhas" icon="bullhorn">
    **Finalidade:** Número máximo de campanhas ativas de chamadas
    
    **Sugestão de Planos Personalizados:**
    - Gratuito: 1 campanha (teste)
    - Básico: 2–3 campanhas
    - Profissional: 5–15 campanhas
    - Enterprise: Ilimitado
  </Accordion>
  
  <Accordion title="Vozes Clonadas" icon="microphone">
    **Finalidade:** Número máximo de vozes personalizadas (clonadas)
    
    **Sugestão de Planos Personalizados:**
    - Gratuito: 0–1 voz (incentivo premium)
    - Básico: 2–3 vozes
    - Profissional: 5–10 vozes
    - Enterprise: Ilimitado
  </Accordion>
  
  <Accordion title="Chamadas Simultâneas" icon="phone">
    **Finalidade:** Quantidade máxima de chamadas paralelas
    
    **Sugestão de Planos Personalizados:**
    - Gratuito: 1–2 chamadas (controle de uso)
    - Básico: 5–10 chamadas
    - Profissional: 20–100 chamadas
    - Enterprise: 500+ chamadas
  </Accordion>
  
  <Accordion title="Bases de Conhecimento" icon="database">
    **Finalidade:** Número máximo de bases de conhecimento
    
    **Sugestão de Planos Personalizados:**
    - Gratuito: 1 base
    - Básico: 2–5 bases
    - Profissional: 10–20 bases
    - Enterprise: Ilimitado
  </Accordion>
  
  <Accordion title="Números Próprios" icon="hashtag">
    **Finalidade:** Quantidade máxima de números telefônicos (SIP / Caller ID)
    
    **Sugestão de Planos Personalizados:**
    - Gratuito: 1 número
    - Básico: 2–3 números
    - Profissional: 5–15 números
    - Enterprise: Ilimitado
  </Accordion>
  
  <Accordion title="Ferramentas Mid-Call" icon="wrench">
    **Finalidade:** Quantidade máxima de ferramentas personalizadas durante a chamada
    
    **Sugestão de Planos Personalizados:**
    - Gratuito: 0 ferramentas (recurso premium)
    - Básico: 1–2 ferramentas
    - Profissional: 5–10 ferramentas
    - Enterprise: Ilimitado
  </Accordion>
  
  <Accordion title="Execuções de Automação" icon="play">
    **Finalidade:** Número mensal de execuções da plataforma de automação
    
    **Sugestão de Planos Personalizados:**
    - Gratuito: 100–500 execuções
    - Básico: 1.000–5.000 execuções
    - Profissional: 10.000–50.000 execuções
    - Enterprise: Ilimitado
  </Accordion>
</AccordionGroup>

### Limites Booleanos
**Controles de acesso a funcionalidades (ativado/desativado):**

<AccordionGroup>
  <Accordion title="Widget Web" icon="code">
    **Finalidade:** Acesso ao widget de atendimento incorporável em sites
    
    **Padrão:** Ativado para todos os planos
  </Accordion>
  
  <Accordion title="Idiomas Secundários" icon="language">
    **Finalidade:** Suporte a múltiplos idiomas nos assistentes de IA
    
    **Padrão:** Ativado para todos os planos
  </Accordion>
  
  <Accordion title="Plataforma de Automação" icon="bolt">
    **Finalidade:** Acesso aos fluxos de automação no-code
    
    **Padrão:** Ativado para todos os planos
  </Accordion>
  
  <Accordion title="Integrações de Calendário" icon="calendar">
    **Finalidade:** Integração com Cal.com e GoHighLevel para agendamentos
    
    **Padrão:** Ativado para todos os planos
  </Accordion>
</AccordionGroup>

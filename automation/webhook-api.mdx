Se você deseja uma personalização mais profunda além dos blocos no-code, a OmniTech suporta **webhooks** e uma **API REST**.

## 1. Webhooks
- **Quando Usar**: Acionar eventos em sistemas externos no momento em que algo acontece na OmniTech (ex.: "Chamada Finalizada").
- **Configuração**: Defina uma URL de webhook nas configurações do seu assistente/campanha ou na plataforma de automação.
- **Payload**: Geralmente inclui ID da chamada, transcrição, resultado da chamada ou quaisquer dados coletados do usuário.

## 2. REST API
- **Principais Funções**:
  1. **Criar/Atualizar Leads**
  2. **Gerenciar Campanhas**
- **Autenticação & Limites de Taxa**: Normalmente utiliza tokens de API. Consulte a [documentação da API](/api-reference/introduction) para detalhes sobre endpoints e cotas de uso.

## 3. Exemplos de Casos de Uso
- **Sincronização em Tempo Real com CRM**: Para cada nova chamada, envie os dados para o endpoint do seu CRM.
- **Dashboards Personalizados**: Extraia dados de chamadas para seu próprio dashboard de análises.

---
**Nota:** Você encontrará a referência oficial da API em uma seção dedicada da documentação ou diretamente no dashboard da OmniTech. Sempre proteja seus tokens e teste os endpoints em um ambiente de desenvolvimento.

Configure o objetivo do assistente e as ferramentas que ele pode utilizar durante as chamadas.

## Base de Conhecimento

Forneça informações para que o assistente utilize como base de conhecimento ao responder clientes, a partir de conteúdos de sites, documentos, entre outros.

### Seleção da Base de Conhecimento

<AccordionGroup>
  <Accordion title="Escolher Base de Conhecimento" icon="book-open">
    Selecione uma base de conhecimento que contenha informações relevantes para o assistente utilizar.
    
    **Requisitos:**
    - Deve ser criada previamente na seção [Bases de Conhecimento](/knowledgebases)
    - Exibe status (Ativa, Processando, etc.) e descrição
    - Apenas bases de conhecimento do próprio usuário ficam disponíveis
    
    <Tip>
    Crie bases de conhecimento a partir de conteúdos de sites, PDFs ou documentos personalizados para fornecer informações específicas do seu domínio ao assistente. Para um guia completo de configuração — incluindo tipos de conteúdo, status de processamento, captura de sites e resolução de problemas — consulte o [Guia Completo de Base de Conhecimento](/conversation-design/knowledge-bases).
    </Tip>
  </Accordion>
  
  <Accordion title="Modo da Base de Conhecimento" icon="brain">
    Escolha como a IA utilizará a base de conhecimento.
    
    <CardGroup cols={2}>
      <Card title="Function Call" icon="function">
        **Recomendado para a maioria dos casos**
        
        Utiliza chamadas de função para buscar informações na base de conhecimento apenas quando necessário, garantindo maior precisão e eficiência.
      </Card>
      
      <Card title="Injeção no Prompt" icon="syringe">
        **Para casos simples**
        
        Realiza buscas após cada fala do cliente. Pode ser mais preciso, porém tende a ser mais lento sem o uso de áudios de preenchimento.
      </Card>
    </CardGroup>
    
    <Note>
    **Disponibilidade do modo:** Os modos Speech-to-Speech e Dualplex suportam apenas o modo Function Call.
    </Note>
  </Accordion>
</AccordionGroup>

## Ferramentas Personalizadas Durante a Chamada

Crie e atribua ferramentas personalizadas que a IA pode utilizar **durante a chamada**, em tempo real com o cliente.

<CardGroup cols={2}>
  <Card title="Criar Ferramentas Personalizadas" icon="plus">
    Desenvolva ferramentas sob medida para suas necessidades de negócio: integrações com APIs, consultas a banco de dados, fluxos personalizados e chamadas a serviços externos.
    
    **Ação rápida:** Link direto para a página de criação de ferramentas.
  </Card>
  
  <Card title="Atribuir ao Assistente" icon="link">

    **Escopo:** Apenas ferramentas do próprio usuário ficam disponíveis.
  </Card>
</CardGroup>

<Tip>
Leia a [Documentação de Ferramentas Personalizadas](/ai-assistants/custom-tools) para guias detalhados de criação e configuração.
</Tip>

## Ferramentas Padrão

Escolha as ferramentas nativas que a IA pode utilizar **durante a chamada** com o cliente.

<AccordionGroup>
  <Accordion title="Transferência Múltipla de Chamadas" icon="phone-arrow-right">
    Direcione chamadas para múltiplos números de destino, como pessoas ou departamentos, de acordo com o fluxo definido.
    
    **Casos de uso:**
    - Escalonamento para atendentes humanos
    - Roteamento por departamento
    - Transferência para especialistas
    
    **Opções de configuração:**
    - **Modo padrão:** Uso de números telefônicos convencionais (ex.: +5511999999999)
    - **Modo avançado:** Uso de URI SIP para provedores SIP (ex.: sip:+5511999999999@sip-server)
    
    **Para integrações SIP:**
    - Defina a transferência como modo "Avançado"
    - Utilize o formato SIP URI: `sip:numero@sip-server`
    - Verifique se seu provedor SIP suporta SIP REFER
    - Teste diferentes formatos de URI se a transferência falhar
    
    <Note>
    **Instrução no prompt:** Inclua no prompt de sistema algo como “chame a função de transferência quando o usuário solicitar falar com outra pessoa” para garantir o uso correto da ferramenta.
    </Note>
    
    <Tip>
    Caso a transferência não funcione, consulte o [Guia de Solução de Problemas de Integração SIP](/troubleshooting/sip-integration#transfer-phone-call-not-working).
    </Tip>
  </Accordion>
  
  <Accordion title="Encerrar Chamada" icon="phone-slash">
    Encerra a chamada quando a conversa for concluída.
    
    **Casos de uso:**
    - Finalização natural da conversa
    - Objetivo atingido
    - Confirmação de satisfação do cliente
    
    <Note>
    **Instrução no prompt:** Caso a IA não encerre a chamada automaticamente, inclua no prompt “chamar a função hang_up para encerrar a chamada” quando apropriado.
    </Note>
  </Accordion>
  
  <Accordion title="Entrada DTMF" icon="hashtag">
    Envie entradas de teclado durante chamadas ativas, permitindo navegação fluida por menus de URA e fluxos automatizados. A IA navega usando botões de teclado conforme as instruções.
    
    **Casos de uso:**
    - Navegação em menus telefônicos
    - Digitação de ramais
    - Inserção de códigos de segurança
    - Interação com sistemas de URA
    
    <Note>
    **Instrução no prompt:** Inclua no prompt quando utilizar a entrada DTMF, como “use a ferramenta send_dtmf para navegar pelos menus telefônicos quando necessário”.
    </Note>
    
    Para mais detalhes sobre DTMF, consulte [Ferramentas e Funções](/ai-assistants/tools-and-functions#4-dtmf-input).
  </Accordion>
  
  <Accordion title="Agendamento de Compromissos" icon="calendar">
    Integração em tempo real para agendamento com múltiplas plataformas de calendário.
    
    **Integrações disponíveis:**
    - **Cal.com:** Guia completo em [Agendamento Cal.com](https://docs.omnitech.ai/ai-assistants/cal-com-scheduling#cal-com-appointment-scheduling)
    - **GoHighLevel:** Guia de integração em [Agendamento GoHighLevel](/ai-assistants/gohighlevel-scheduling)
    
    **Casos de uso:**
    - Agendar compromissos automaticamente
    - Consultar disponibilidade em tempo real
    - Enviar confirmações automáticas
    - Sincronizar com calendários existentes
    
    <Note>
    Disponível quando a integração de calendário estiver habilitada nas configurações da plataforma.
    </Note>
  </Accordion>
</AccordionGroup>

## Variáveis de Chamada

Defina variáveis que podem ser passadas **antes** da chamada e utilizadas no prompt no formato **{nome_da_variavel}**.

### Configuração de Variáveis

<AccordionGroup>
  <Accordion title="Definição de Variáveis" icon="code">
    Crie variáveis de chamada para fornecer informações ao assistente antes do início da chamada.
    
    **Processo de configuração:**
    - Defina o nome e valor padrão das variáveis
    - Passe valores ao criar chamadas, leads ou importar clientes
    - As variáveis ficam disponíveis imediatamente nos prompts e durante a chamada
    - Use para personalização e contextualização
    
    **Configuração:**
    - **Nome da variável:** Identificador (ex.: `customer_name`)
    - **Valor padrão:** Utilizado quando não for informado
  </Accordion>
  
  <Accordion title="Uso das Variáveis" icon="code">
    Como utilizar variáveis de chamada no assistente:
    
    **Nos prompts de sistema:**
    - Referencie usando `{nome_da_variavel}`
    - Personalize conversas dinamicamente
    - Forneça contexto específico
    
    **Exemplos:**
    - `{customer_name}` para personalização
    - `{email}` para integrações de calendário
    - `{account_type}` para respostas personalizadas
    - `{company}` para contexto empresarial
  </Accordion>
  
  <Accordion title="Fontes de Dados" icon="database">
    Formas de alimentar variáveis de chamada:
    
    **Entrada manual:**
    - Ao criar chamadas individuais
    - Durante a criação de leads em campanhas
    
    **Métodos de importação:**
    - Importação via CSV
    - Integração via API
    - Sincronização direta com CRM
    
    **Integrações com plataformas de automação:**
    - **GoHighLevel (GHL):** Importação automática de dados e campos personalizados
    - **Google Sheets:** Importação de leads por planilhas
    - **Conexão com mais de 250 ferramentas** via plataforma no-code
    - **Qualquer outra plataforma** via webhooks personalizados
    
    **Integração com a plataforma:**
    - Preenchimento automático por perfis de clientes
    - Conexão com bancos de dados existentes
    - Fluxos de automação sem código
  </Accordion>
</AccordionGroup>

<Note>
As variáveis de chamada são essenciais para personalizar conversas com IA, tornando as interações mais relevantes e eficazes.
</Note>

## Configuração do Fluxo de Chamada

### Quem Fala Primeiro?

Defina se o assistente ou o cliente inicia a conversa.

<CardGroup cols={2}>
  <Card title="Assistente de IA" icon="robot">
    **Comportamento padrão:** O assistente inicia a conversa
    
    A IA cumprimenta o cliente assim que a chamada é conectada.
    
    **Ideal para:** Chamadas ativas, vendas e lembretes
  </Card>
  
  <Card title="Cliente" icon="user">
    **Aguardar cliente:** O assistente escuta primeiro
    
    A IA aguardará o cliente falar antes de responder.
    
    **Ideal para:** Chamadas receptivas, suporte e recepção
  </Card>
</CardGroup>

### Mensagem Inicial

Configure a primeira mensagem que o assistente dirá ao iniciar a chamada.

<AccordionGroup>
  <Accordion title="Texto da Mensagem Inicial" icon="message">
    Defina a mensagem de abertura que o assistente falará no início da chamada.
    
    **Boas práticas:**
    - Seja objetivo e cordial
    - Apresente sua empresa e propósito
    - Defina expectativas
    - Utilize variáveis para personalização
    
    **Exemplo:** "Olá {customer_name}, aqui é a OmniTech. Estou entrando em contato para dar continuidade à sua solicitação. Podemos conversar agora?"
  </Accordion>
  
  <Accordion title="Arquivo de Áudio Inicial" icon="volume">
    **Opcional:** Envie um áudio personalizado no lugar do texto
    
    **Requisitos:**
    - Formato MP3 ou WAV
    - Áudio claro e profissional
    - Tom compatível com a voz do assistente
    
    **Benefícios:**
    - Pronúncia perfeita
    - Padronização da marca
    - Primeira impressão profissional
    - Sem latência de TTS
    
    <Tip>
    Funciona melhor quando a voz da IA é clonada a partir do mesmo locutor do áudio inicial.
    </Tip>
  </Accordion>
</AccordionGroup>

<Note>
Consulte o [Guia de Mensagem Inicial](/ai-assistants/initial-message) para exemplos e boas práticas.
</Note>

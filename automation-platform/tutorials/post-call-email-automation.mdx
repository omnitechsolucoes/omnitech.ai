Configurar modelos de e-mail com conteúdo dinâmico das chamadas

Pré-requisitos

Assistente de IA configurado com variáveis pós-chamada e coleta de e-mail

Entendimento de automação via webhook pós-chamada

Acesso à funcionalidade de testes de chamadas

Etapa 1: Criar automação de e-mail pós-chamada

Vamos criar uma automação que será acionada após chamadas telefônicas e enviará e-mails personalizados com base nos resultados da chamada.

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/01-create-email-automation.mp4"></video>

Acesse a Plataforma de Automação

Crie um novo fluxo do zero

Pesquise pela sua plataforma de chamadas nos gatilhos

Selecione o gatilho "Phone Call Ended"

Isso captura dados de todas as chamadas concluídas do assistente selecionado

Etapa 2: Configurar o assistente e a integração de webhook

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/03-select-assistant-for-email.mp4"></video>

Selecione o assistente desejado

Cada assistente suporta apenas uma automação pós-chamada

Todas as chamadas desse assistente acionarão este fluxo de e-mail

Clique em "Test Trigger" para configurar o webhook

Isso gera e salva automaticamente a URL do webhook

O webhook é armazenado nas configurações pós-chamada do assistente

Etapa 3: Verificar a configuração do assistente

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/05-check-assistant-webhook.mp4"></video>

Abra o assistente em uma nova aba

Acesse a aba "Post-call Actions" (terceira aba)

Confirme que a URL do webhook foi configurada automaticamente

<Frame> <img src="/resources/tutorials/post-call-email/06-webhook-settings.png" alt="Configuração do gatilho de webhook" /> </Frame>

Revise as configurações do gatilho:

Enabled: A automação será acionada após as chamadas

Completed calls only: Recomendado para e-mails de follow-up

All calls: Selecione "No" para incluir chamadas com falha

<Frame> <img src="/resources/tutorials/post-call-email/07-post-call-variables.png" alt="Variáveis pós-chamada para conteúdo de e-mail" /> </Frame>

Verifique as variáveis pós-chamada:

status: Booleano indicando sucesso da chamada

email: E-mail do cliente (garanta que seja coletado durante a chamada)

Etapa 4: Garantir a coleta de e-mail

Para que a automação de e-mail funcione, o assistente precisa coletar os endereços de e-mail dos clientes.

<Frame> <img src="/resources/tutorials/post-call-email/08-set-test-values.png" alt="Variáveis de chamada incluindo coleta de e-mail" /> </Frame>

Verifique as variáveis de chamada do assistente:

Garanta que a variável email esteja definida

Defina um e-mail padrão para testes. Use seu e-mail real para testar

Considere tornar a coleta de e-mail parte do roteiro da chamada

Etapa 5: Gerar chamada de teste com dados de e-mail

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/09-make-test-call.mp4"></video>

Faça uma chamada de teste para gerar dados do webhook:

Clique em "Speak to Assistant" → "Phone Call"

Atenda quando o assistente ligar

Responda positivamente para garantir que a variável status seja true

Conclua o objetivo da chamada durante a conversa

Finalize a chamada normalmente

<Note> Em um cenário real, o assistente utilizará os dados atribuídos ao lead. Para testes, ele usa os valores padrão configurados. </Note>

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/10-check-call-results.mp4"></video>

Verifique os resultados da chamada e a extração das variáveis:

Acesse a lista de chamadas e localize a chamada recente

Clique na chamada para ver os detalhes

Confirme que a variável status está como true

As variáveis são extraídas com base em:

Transcrição da chamada

Descrições das variáveis

Tipos das variáveis (Boolean para status)

Contexto do system prompt

<Note> As variáveis pós-chamada são extraídas após o término da chamada. Atualize a página após cerca de um minuto para visualizá-las. </Note>

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/11-received-call-data.mp4"></video>

Retorne à aba da automação

Verifique se os dados da chamada foram recebidos

Os dados aparecem porque você clicou em "Test Trigger" e realizou a chamada

A automação recebeu a requisição após o término da chamada e extração das variáveis

Método alternativo – reenviar dados de webhook de chamadas existentes:

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/12-resend-webhook.mp4"></video>

Caso precise testar com dados diferentes, reutilize qualquer chamada existente

Clique novamente em "Test Trigger"

Acesse uma chamada concluída

Clique em "Resend Webhook"

Retorne à aba da automação para ver os dados atualizados

Adicionando novas variáveis pós-chamada
<Note> Se precisar adicionar uma nova variável pós-chamada após criar a automação, siga este processo para atualizar os dados. </Note>

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/add-variable.mp4"></video>

Defina a nova variável no assistente:

Vá em configurações do assistente → aba "Post-call Actions"

Adicione a nova variável (nome, tipo, descrição)

Salve o assistente

Atualize a automação para receber a nova variável:

Retorne à automação

Clique em "Test Trigger"

Reavaliar uma chamada existente:

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/re-evaluate.mp4"></video>

Abra uma chamada concluída desse assistente

Clique em "Re-evaluate"

Aguarde o processamento

Enviar dados atualizados para a automação:

Clique em "Resend Webhook"

Retorne à automação

Confirme que a nova variável está disponível

<Tip> O recurso de reenvio é útil para testar diferentes cenários sem realizar novas chamadas. </Tip>
Etapa 6: Configurar lógica condicional de e-mail

Queremos enviar e-mails apenas quando o objetivo da chamada for alcançado.

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/13-add-branch.mp4"></video>

Clique no botão "+" para adicionar uma etapa

Pesquise por "Branch" no controle de fluxo

Selecione Branch para criar a lógica condicional

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/14-configure-status-condition.mp4"></video>

Configure a condição do branch:

Selecione a variável status

Escolha "Boolean is true"

Isso cria dois caminhos: True (qualificado) e False (não qualificado)

Etapa 7: Configurar integração com serviço de e-mail

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/15-add-email-action.mp4"></video>

No branch True, clique em "+"

Pesquise pelo serviço de e-mail (Gmail, Outlook, SMTP, etc.)

Selecione a ação "Send Email"

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/16-create-email-connection.mp4"></video>

Crie a conexão com o serviço de e-mail:

Autorize o provedor

Configure SMTP se necessário

Salve a conexão

Etapa 8: Configurar conteúdo e destinatários do e-mail

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/17-map-recipient-email.mp4"></video>

Configurar destinatário:

Clique no campo "To"

Selecione o e-mail do cliente a partir das variáveis

Garanta que o e-mail esteja corretamente mapeado

Definir assunto do e-mail:

Use conteúdo dinâmico: "Obrigado pelo seu tempo, {customer_name}!"

Inclua o nome da empresa ou referência da chamada

<video autoPlay muted loop playsinline src="/resources/tutorials/post-call-email/19-compose-email-body.mp4"></video>

Criar o corpo do e-mail com conteúdo dinâmico:

Olá {customer_name},

Obrigado por dedicar seu tempo para falar conosco hoje. Abaixo está um resumo da nossa conversa:

{formatted_transcript}

Próximos passos:
- Faremos o follow-up conforme combinado
- Fique à vontade para entrar em contato em caso de dúvidas
- [Inclua os próximos passos específicos do seu negócio]

Atenciosamente,
[Seu Nome/Empresa]
[Informações de Contato]

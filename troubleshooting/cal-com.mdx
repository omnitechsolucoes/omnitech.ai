Soluções para problemas comuns de integração com o Cal.com.

## Problemas de Conexão

### API Key Inválida

**Problema:** Eventos não carregam ou a sincronização falha

**Solução:**
- Verifique se a API key está ativa em Cal.com → Settings → Developer → API Keys
- Confirme se a chave contém o prefixo `cal_live_`
- Gere uma nova API key, se necessário

### Eventos Não Carregam

**Problema:** O dropdown de eventos fica vazio após inserir a API key

**Solução:**
- Verifique se a API key está correta
- Clique fora do campo da API key para disparar o carregamento dos eventos
- Atualize a página se os eventos não aparecerem
- Confirme se existem eventos criados no Cal.com

## Problemas de Sincronização

### Falha ao Sincronizar Evento

**Problema:** O botão "Sync Event" falha ou exibe erro

**Solução:**
1. Clique no botão **"Troubleshoot"**
2. Isso redefine todos os campos de agendamento para a configuração correta
3. Verifique as alterações nas configurações do evento no Cal.com

### Erros de Configuração de Campos

**Problema:** Erros sobre campos obrigatórios ou campos ausentes

**Solução:**
- Clique em **"Troubleshoot"** para redefinir os campos
- Apenas nome, e-mail e telefone devem ser obrigatórios
- Todos os outros campos (localização, convidados, etc.) devem ser opcionais

## Falhas de Agendamento

### Convites de Calendário Não Enviados

**Problema:** O agendamento é concluído, mas o cliente não recebe o convite

**Solução:**
- Defina a variável `email` nas [configurações do assistente](/ai-assistants/settings/prompt-and-tools#call-variables)
- Envie o e-mail do cliente ao criar os leads
- Verifique se o formato do e-mail é válido

### Número de Telefone Ausente

**Problema:** O número de telefone não aparece nos eventos do calendário

**Solução:**
- Clique em **"Sync Event"** ou **"Troubleshoot"**
- Isso adiciona automaticamente o campo de telefone ao evento
- O número passa a ser incluído após a sincronização

### Problemas de Fuso Horário no Agendamento

**Problema:** Compromissos são agendados em horário incorreto

**Solução:**
- Verifique o fuso horário nas configurações do evento no Cal.com
- Confirme se o assistente está recebendo o horário correto do cliente
- Teste mencionando horários específicos durante a chamada

## Problemas de Testes

### Agendamentos via Web Call Falham

**Problema:** Agendamentos funcionam de forma inconsistente com chamadas via web

**Solução:**
- Utilize chamadas telefônicas reais para testes (não chamadas web)
- Chamadas web podem não enviar corretamente a variável de telefone
- Ao usar chamadas web, informe o telefone no formato internacional com prefixo + (ex.: "+1 555-123-4567")
- Chamadas telefônicas garantem que todas as variáveis sejam transmitidas corretamente

### Múltiplos Calendários Não Funcionam

**Problema:** Apenas um calendário funciona ou há conflito entre eventos

**Solução:**
- Cada calendário deve ter um evento diferente selecionado
- Utilize nomes de eventos distintos (ex.: "15min", "30min")
- Sincronize cada evento separadamente após adicioná-lo

## Obtendo Ajuda

Se os problemas persistirem:

1. Verifique as transcrições das chamadas para tentativas de agendamento
2. Confirme se o evento aparece no painel do Cal.com
3. Teste com um agendamento simples via chamada telefônica
4. Entre em contato com o suporte informando:
   - Captura de tela da mensagem de erro
   - Captura de tela das configurações do evento no Cal.com
   - Transcrição da chamada mostrando a tentativa de agendamento

A plataforma combina **reconhecimento de voz por IA** com **respostas baseadas em LLM** para conduzir conversas telefônicas em tempo real. Abaixo está um fluxo simplificado do funcionamento:

## Fluxo Geral

1. **Chamada Iniciada (Caller ou Lead)**

   - **Inbound:** Um cliente liga para o número telefônico atribuído.
   - **Outbound:** O sistema realiza chamadas para uma lista de contatos a partir de uma campanha.

2. **Reconhecimento de Fala**

   - A IA escuta o que a pessoa diz.
   - O áudio é transcrito para texto em tempo real.

3. **Decisão pelo LLM**

   - O modelo de linguagem (LLM) analisa o texto.
   - O **system prompt** define o tom, as regras e a próxima ação (responder, transferir, encerrar, etc.).

4. **Geração de Voz**

   - A resposta é convertida em áudio natural por meio do motor de voz (ex.: vozes ElevenLabs).

5. **Resultado e Ações**

   - **Inbound:** A IA responde dúvidas, valida informações ou registra recados.
   - **Outbound:** A IA apresenta ofertas, qualifica leads ou transfere para um atendente humano.

6. **Dados e Relatórios**

   - Transcrição completa da chamada
   - Gravação de áudio
   - Status da chamada (sucesso, não atendida, falha, etc.)
   - Possibilidade de automações pós-chamada (CRM, e-mail, planilhas)

## Diferenças Entre Inbound e Outbound

### Inbound

- O cliente liga para um número alugado ou integrado via SIP.
- A IA atende automaticamente.
- Segue o prompt configurado para suporte, vendas ou triagem.
- Pode transferir para um humano se necessário.

### Outbound

- Leads são importados ou sincronizados.
- A IA liga automaticamente para cada contato.
- Segue um roteiro definido no prompt.
- Pode transferir chamadas qualificadas para agentes humanos.

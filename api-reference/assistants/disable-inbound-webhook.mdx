---
title: "Desativar webhook inbound do assistente"
api: "POST /user/assistants/disable-inbound-webhook"
description: "Desativa as notificações de webhook inbound para um assistente específico"
---

Este endpoint desativa as notificações de webhook inbound de um assistente, interrompendo o envio de atualizações em tempo real sobre conclusões de chamadas inbound e dados.

### Corpo da Requisição

<ParamField body="assistant_id" type="integer" required>
  O ID do assistente para o qual o webhook será desativado
</ParamField>

### Resposta

<ResponseField name="message" type="string">
  Mensagem de sucesso confirmando que o webhook foi desativado
</ResponseField>

<ResponseField name="data" type="array">
  Array vazio (reservado para uso futuro)
</ResponseField>

### Respostas de Erro

<ResponseField name="404 Not Found">
  <Expandable title="Resposta de Erro">
    <ResponseField name="message" type="string">
      Mensagem de erro quando o assistente não é encontrado ou não pertence ao usuário autenticado
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="422 Validation Error">
  <Expandable title="Resposta de Erro">
    <ResponseField name="message" type="string">
      Mensagem de erro indicando falha de validação
    </ResponseField>
    <ResponseField name="errors" type="object">
      Erros detalhados de validação para cada campo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

```json 200 Success Response
{
  "message": "Webhook inbound desativado com sucesso",
  "data": []
}
{
  "message": "Assistente não encontrado"
}
{
  "message": "Os dados fornecidos são inválidos.",
  "errors": {
    "assistant_id": [
      "O campo ID do assistente é obrigatório."
    ]
  }
}
</ResponseExample> ```

---
title: "Ativar webhook inbound do assistente"
api: "POST /user/assistants/enable-inbound-webhook"
description: "Ativa as notificações de webhook inbound para um assistente específico"
---

Este endpoint ativa as notificações de webhook inbound de um assistente, permitindo o recebimento de atualizações em tempo real sobre conclusões de chamadas inbound e dados.

### Corpo da Requisição

<ParamField body="assistant_id" type="integer" required>
  O ID do assistente para o qual o webhook será ativado
</ParamField>

<ParamField body="webhook_url" type="string" required>
  A URL para onde as notificações de webhook serão enviadas
</ParamField>

### Resposta

<ResponseField name="message" type="string">
  Mensagem de sucesso confirmando que o webhook foi ativado
</ResponseField>

<ResponseField name="data" type="array">
  Array vazio (reservado para uso futuro)
</ResponseField>

### Respostas de Erro

<ResponseField name="404 Not Found">
  <Expandable title="Resposta de Erro">
    <ResponseField name="message" type="string">
      Mensagem de erro quando o assistente não é encontrado ou não pertence ao usuário autenticado
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="422 Validation Error">
  <Expandable title="Resposta de Erro">
    <ResponseField name="message" type="string">
      Mensagem de erro indicando falha de validação
    </ResponseField>
    <ResponseField name="errors" type="object">
      Erros detalhados de validação para cada campo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

```json 200 Success Response
{
  "message": "Webhook inbound ativado com sucesso",
  "data": []
}
{
  "message": "Assistente não encontrado"
}
{
  "message": "Os dados fornecidos são inválidos.",
  "errors": {
    "assistant_id": [
      "O campo ID do assistente é obrigatório."
    ],
    "webhook_url": [
      "O campo URL do webhook é obrigatório."
    ]
  }
}
</ResponseExample> ```
